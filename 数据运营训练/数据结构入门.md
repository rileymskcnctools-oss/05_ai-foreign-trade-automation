# 数据运营能力训练

> 结合外贸场景学习数据结构
> 不需要编程背景，用外贸思维理解

## 一、为什么外贸人需要懂数据？

传统外贸 vs AI 外贸的区别：

```
传统外贸：
  产品表（纸质/Excel）→ 人工写文案 → 发布
  ↓
  慢、容易出错、难以批量

AI 外贸：
  产品表（结构化数据）→ AI 自动处理 → 批量发布
  ↓
  快、准确、可批量
```

关键区别：**数据结构化**。
AI 只能处理结构化的数据，不能处理模糊的描述。

## 二、CSV — 最简单的数据格式

### 什么是 CSV？

CSV = Comma-Separated Values（逗号分隔的值）

它本质上是**纯文本**，用逗号分隔每一列的数据。

### Excel vs CSV

```
Excel (.xlsx)：
  - 有格式（颜色、字体、合并单元格）
  - AI 理解起来复杂
  - 适合给人看

CSV (.csv)：
  - 纯文本，没有格式
  - AI 理解起来简单
  - 适合给机器处理
```

### 示例

Excel 中的产品表：
```
| product_id | product_name_en | material      | weight_kg |
|-----------|----------------|---------------|-----------|
| GF-001    | Garden Fork    | Carbon Steel  | 0.8       |
| GS-001    | Garden Spade   | Carbon Steel  | 0.6       |
```

同样的数据，CSV 格式：
```csv
product_id,product_name_en,material,weight_kg
GF-001,Garden Fork,Carbon Steel,0.8
GS-001,Garden Spade,Carbon Steel,0.6
```

### 外贸场景理解

**CSV 就像一份简化的订单表**：
- 每一行 = 一个产品
- 每一列 = 一种信息
- 用逗号分隔，方便机器读取

### 实操：Excel 转 CSV

1. 打开你的产品 Excel 表
2. 点击"文件" → "另存为"
3. 选择保存类型为 "CSV (逗号分隔)"
4. 保存

### 注意事项

- CSV 不支持合并单元格
- CSV 不支持公式
- CSV 不支持多个 Sheet
- 如果内容中有逗号，需要用引号包裹：`"Carbon Steel, Ash Wood"`

## 三、JSON — AI 最喜欢的数据格式

### 什么是 JSON？

JSON = JavaScript Object Notation

它是**键值对**结构的数据格式。AI 特别喜欢 JSON，因为：
- 结构清晰
- 可以嵌套（一个值里面还可以有值）
- 容易解析

### 示例

同一个产品的数据，JSON 格式：
```json
{
  "product_id": "GF-001",
  "product_name_en": "Garden Fork",
  "material": "Carbon Steel",
  "handle_material": "Ash Wood",
  "specifications": {
    "length_cm": 30,
    "weight_kg": 0.8,
    "tine_count": 4
  },
  "target_markets": ["Nigeria", "Germany", "Brazil"]
}
```

### 外贸场景理解

**JSON 就像一份详细的客户档案**：

```
客户：John Smith
├── 基本信息
│   ├── 公司名：Green Gardens Ltd
│   ├── 国家：Nigeria
│   └── 联系方式：+234 xxx xxx xxxx
├── 采购记录
│   ├── 2024年：Garden Fork × 2000 pcs
│   └── 2025年：Garden Spade × 1500 pcs
└── 偏好
    ├── 包装：Color Box
    └── 关注点：价格、交期
```

用 JSON 写出来：
```json
{
  "customer": "John Smith",
  "basic_info": {
    "company": "Green Gardens Ltd",
    "country": "Nigeria",
    "phone": "+234 xxx xxx xxxx"
  },
  "purchase_history": [
    {"year": 2024, "product": "Garden Fork", "qty": 2000},
    {"year": 2025, "product": "Garden Spade", "qty": 1500}
  ],
  "preferences": {
    "packaging": "Color Box",
    "focus": ["price", "lead_time"]
  }
}
```

### JSON 的基本语法

| 符号 | 含义 | 示例 |
|------|------|------|
| `{ }` | 对象（一组键值对） | `{"name": "Garden Fork"}` |
| `[ ]` | 数组（一组值） | `["Nigeria", "Germany"]` |
| `" "` | 字符串（文字） | `"Carbon Steel"` |
| 数字 | 数字（不需要引号） | `0.8`, `500` |
| `:` | 键和值的分隔 | `"name": "Garden Fork"` |
| `,` | 多个值的分隔 | `"a": 1, "b": 2` |

### 为什么 AI 输出用 JSON 最好？

1. **结构化**：AI 返回的内容有明确的标签
2. **可解析**：程序可以自动提取需要的部分
3. **可嵌套**：复杂数据也能表达

对比：

```
❌ AI 返回文本：
SEO 标题有 5 个。第一个是...第二个是...第三个是...
（你需要自己提取）

✅ AI 返回 JSON：
{
  "seo_titles": [
    {"variant": 1, "title": "..."},
    {"variant": 2, "title": "..."},
    ...
  ]
}
（程序可以直接提取每个标题）
```

## 四、Excel — 你最熟悉的工具

### Excel 在系统中的角色

Excel 是**你的输入界面**——你用 Excel 录入产品信息。

### Excel vs CSV vs JSON 的分工

```
Excel（你录入数据）
    ↓
    导出为 CSV（给机器处理）
    ↓
    AI 处理，返回 JSON 结果
    ↓
    JSON 结果导出为 Excel/CSV（你查看结果）
```

### Excel 使用技巧（外贸场景）

1. **不要合并单元格**
   - 合并单元格会破坏数据结构
   - AI 无法正确读取合并的单元格

2. **第一行是标题行**
   - 每列的第一行是字段名
   - 用英文字段名（AI 理解更准确）

3. **每行一个产品**
   - 不要把多个产品的信息放在同一行

4. **用数据验证创建下拉菜单**
   - 类别、包装方式等字段用下拉菜单
   - 避免输入不一致

5. **冻结首行**
   - 方便查看大量数据时，始终看到列名

## 五、基本数据结构概念

### 1. 行（Row）= 一条记录

在 Excel 中：
- 一行 = 一个产品的所有信息
- 一行 = 一个客户的所有信息
- 一行 = 一个订单的所有信息

```
第 2 行：
| GF-001 | Garden Fork | Carbon Steel | 30 | 0.8 | 4 | ...
          ↑ 一个产品的完整信息
```

### 2. 列（Column）= 一个字段

- 一列 = 一种信息
- 一列 = 一个属性

```
第 F 列：material（材质）
| material        |
|-----------------|
| Carbon Steel    | ← 产品 A 的材质
| Carbon Steel    | ← 产品 B 的材质
| Stainless Steel | ← 产品 C 的材质
```

### 3. 表（Table）= 所有记录

- 一张表 = 一类数据
- 例如：产品表、客户表、订单表

### 4. 关系（Relationship）

表与表之间的关系：

```
产品表                    订单表
┌──────┬───────────┐     ┌──────┬───────────┬──────┐
| ID   | 产品名     |     | ID   | 产品ID    | 数量  |
├──────┼───────────┤     ├──────┼───────────┼──────┤
| GF-001| Garden Fork|     | 001  | GF-001    | 2000 |
| GS-001| Garden Spade|     | 002  | GF-001    | 500  |
└──────┴───────────┘     | 003  | GS-001    | 1000 |
                          └──────┴───────────┴──────┘

通过 product_id 关联两个表
```

### 外贸场景理解

这就像你的业务台账：
- **产品表**：你卖的所有产品
- **客户表**：你的所有客户
- **订单表**：每个订单买了什么

通过"产品编号"和"客户编号"，你可以：
- 查某个产品被哪些客户买过
- 查某个客户买了哪些产品

## 六、数据清洗基础

### 什么是数据清洗？

在把数据交给 AI 之前，确保数据是"干净"的。

### 常见问题

| 问题 | 示例 | 修正 |
|------|------|------|
| 空值 | 某行没有填材质 | 补充数据或标记为 "N/A" |
| 格式不一致 | "碳钢" / "carbon steel" / "CS" | 统一为 "Carbon Steel" |
| 多余空格 | " Garden Fork " | 去掉前后空格 |
| 错误数据 | 重量写了 "800g"（应该写 0.8） | 统一单位 |

### 数据清洗检查清单

- [ ] 所有必填字段都有值
- [ ] 同一字段的格式一致
- [ ] 数字字段只包含数字
- [ ] 日期字段格式统一（如 YYYY-MM-DD）
- [ ] 没有多余的空格

## 八、关键术语速查

| 术语 | 含义 | 外贸比喻 |
|------|------|---------|
| 字段（Field） | 表格的一列 | 一种信息类型（如"材质"） |
| 记录（Record） | 表格的一行 | 一个产品的完整信息 |
| 值（Value） | 某个单元格的内容 | 具体数据（如"碳钢"） |
| 键（Key） | JSON 中的标签 | 信息的名称 |
| 数组（Array） | 一组值 | 一个产品的多个关键词 |
| 对象（Object） | 一组键值对 | 一个产品的完整档案 |
| 数据清洗 | 整理数据 | 整理产品目录 |
| 数据结构 | 数据的组织方式 | 产品表的列和行的安排 |
