### 1.3 v1.0 工作流程（手动操作）

1. 打开 CSV 文件 → 找到产品型号
2. 手动复制产品数据 → 粘贴到 Prompt 模板的占位符
3. 手动调用 AI（Qwen）→ 获取生成结果
4. 手动复制结果 → 保存到对应文件夹
5. 每个产品重复以上步骤

### 1.4 v1.0 的核心问题

| 问题 | 影响 |
|------|------|
| 数据存在CSV里 | 无法搜索/筛选/统计，只能打开Excel一行行找 |
| Prompt 文件含大量示例说明 | 发给AI的内容太长，浪费token，响应慢 |
| 手动复制粘贴 | 每个产品要操作5-10分钟，215个产品 = 几十小时 |
| 无数据库 | 生成的内容散落各处，无法追踪哪些产品已生成 |
| 无批量能力 | 只能一次做一个产品 |
| 无客户管理 | 客户信息在聊天记录里，无法搜索和跟进 |

### 2.3 v1.0 vs v2.0 对比

| 维度 | v1.0 | v2.0 |
|------|------|------|
| 数据存储 | CSV文件（Excel手动操作） | SQLite数据库（程序化读写） |
| Prompt | 含示例说明的文档，需手动替换 | 清理后纯模板，自动填充 |
| 生成方式 | 手动复制粘贴 → 调用AI → 保存 | 一条命令自动完成全流程 |
| 产品搜索 | 打开Excel → Ctrl+F | `ft_cli.py search hoe` |
| 批量能力 | 无 | 支持批量生成（开发中） |
| 生成状态追踪 | 不知道哪些产品已做 | 数据库字段标记状态 |
| 多模型支持 | 仅Qwen | Qwen/OpenAI/Claude/Gemini 可路由 |
| 配置文件 | 手动写在代码里 | settings.yaml + .env 分离 |
| 换电脑 | 手动重建环境 | `git clone` + 迁移脚本一键恢复 |

## 三、数据迁移详情

### 3.1 CSV → SQLite 迁移结果

- **源文件**：`04-database/csv/product_database_filled.csv`（GBK编码，31列，216行含表头）

- **迁移结果**：215条产品记录，0丢失，0错误
- **字段映射**：31列全部按 `field_mapping.md` 映射到数据库字段

### 3.2 数据完整性

| 类别 | 数量 |

| 总产品 | 215 |
| Weeding Tools（除草） | 81 |
| Digging Tools（挖掘） | 74 |

## 五、如何衔接到 v2.0 日常工作

### 5.1 过渡期策略：v1.0 和 v2.0 并行

不要立刻放弃 v1.0。过渡期这样做：

**继续使用 v1.0 的场景：**
- 生成市场研究报告（v2.0还没做M4）
- 生成RFQ回复（v2.0还没做M6）
- 任何v2.0还没实现的模块

**开始使用 v2.0 的场景：**
- 产品搜索：`python scripts/ft_cli.py search hoe`
- 查看产品详情：`python scripts/ft_cli.py get GF-001`
- 生成SEO标题：`python scripts/ft_cli.py generate GF-001`（待开发）
- 导出数据：`python scripts/ft_cli.py export --category "Weeding Tools"`

### 5.2 日常操作对照表

| 你想做的事 | v1.0 做法 | v2.0 做法 |
|------------|-----------|-----------|
| 查找某个产品 | 打开Excel，Ctrl+F搜索 | `ft_cli.py search hoe` |
| 查看产品完整信息 | 在Excel里左右滚动看31列 | `ft_cli.py get GF-001` |
| 生成SEO标题 | 复制数据→改Prompt→调AI→保存 | `ft_cli.py generate GF-001 --type seo` |
| 生成卖点 | 同上 | `ft_cli.py generate GF-001 --type selling_points` |
| 查看哪些产品缺数据 | 手动检查Excel列 | `ft_cli.py missing` |
| 导出所有除草工具 | Excel筛选→复制→粘贴 | `ft_cli.py export --category "Weeding Tools"` |

### 5.3 换电脑/换环境操作

如果你要在另一台电脑上工作：

```bash
# 1. 克隆仓库
git clone https://github.com/rileymskcnctools-oss/05_ai-foreign-trade-automation.git
cd 05_ai-foreign-trade-automation

# 2. 复制v1.0的CSV文件（如果没有的话）
# 确保 04-database/csv/product_database_filled.csv 存在

# 3. 配置API key
cp config/.env.example config/.env
# 编辑 config/.env，填入你的 DASHSCOPE_API_KEY

# 4. 运行迁移脚本（创建SQLite数据库，导入CSV数据）
python scripts/migrate_v1_to_v2.py

# 5. 验证环境
python scripts/verify_setup.py

# 6. 使用CLI
python scripts/ft_cli.py help
```
